# Design Context — marriage-system

This file gates UI work. The full design context lives in:

- **PRODUCT.md** — register (product), users (the couple, mobile-first), purpose (wedding cockpit: vendors, budget, payments, checklist), brand personality (elegante, acolhedor, preciso), anti-references (cold spreadsheets, generic SaaS dashboards, wedding kitsch, no copying WeddingWire/The Knot/Zola/Joy/iCasei visuals), design principles, accessibility (AA, status ≠ color alone, 44px targets).
- **DESIGN.md** — warm ivory + espresso + sage-olive palette with champagne gold for celebration; Fraunces display + Geist body; tabular numbers; rounded-2xl cards; bottom tab bar (mobile) / sidebar (desktop); pt-BR, BRL, dd/MM/yyyy.

Read both before any frontend change. Tokens are defined in `app/globals.css` — never hardcode hex in components.
